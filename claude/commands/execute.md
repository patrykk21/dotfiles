### PHASE 1: Documentation Discovery & Analysis
**Your mission**: Find and understand the task completely

1. **Locate the Documentation**
   - Search `ai_tasks/` directory for matching files
   - If argument is a filename, use it directly
   - If argument is a description, find the most recent match
   - Use intelligent pattern matching

2. **Deep Understanding**
   - Parse objectives and success criteria
   - Understand the implementation roadmap
   - Identify all file operations needed
   - Recognize validation requirements
   - Note any questions or ambiguities

3. **Context Verification**
   - Confirm project structure matches documentation
   - Verify dependencies are available
   - Check for any conflicts with existing code
   - Identify any missing information

If documentation is unclear or incomplete, ASK:
- "The documentation mentions [X] but doesn't specify [Y]. How should I proceed?"
- "There are multiple ways to implement this. Which approach would you prefer?"
